Optimization

# Gradient Descent

Core idea: Iteratively move parameters in the direction of the negative gradient to minimize a loss function.

θ:=θ−η⋅∇θ​J(θ)
Interactive · gradient_descent_interactive.html

# Variants

Variant Batch Size Pros Cons
Batch GD Full dataset Stable, exact gradient Slow, memory-heavy
Stochastic (SGD) 1 sample Fast updates, escapes local minima Noisy, high variance
Mini-batch GD n samples Balance of both Needs tuning of batch size

# Key Hyperparameters

  • Learning rate η — most critical; too large → diverge, too small → slow convergence
  • Batch size — larger = smoother gradient, less noise; smaller = more regularization effect
  • Momentum — accumulates past gradients: v = βv + η∇J, reduces oscillation

# Common Optimizers

Optimizer Key Idea
SGD + Momentum Dampens oscillations, accelerates relevant directions
RMSProp Adapts lr per-param using running avg of squared gradients
Adam Momentum + RMSProp; first & second moment estimates
AdamW Adam + decoupled weight decay (preferred in practice)

Adam update:

θ:=θ−v^​+ϵη​m^

# Challenges

  • Saddle points — more common than local minima in high-dim; SGD noise helps escape
  • Vanishing/exploding gradients — addressed by normalization, gradient clipping, careful init
  • Learning rate schedule — step decay, cosine annealing, warmup + decay

# ⚠️ Tricky Interview Facts

  1. Local minima are rarely the real problem in deep networks — saddle points and plateaus are. Most local minima in high-dim spaces have loss values close to the global minimum.
  2. Larger batch size ≠ always better — empirically, small batches act as implicit regularizers (sharp vs. flat minima); flat minima generalize better (Keskar et al., 2017).
  3. Adam doesn’t always generalize as well as SGD — Adam converges faster but SGD with tuned lr/momentum often reaches lower test loss; common to switch to SGD late in training.
