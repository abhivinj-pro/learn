Fundamental Concepts

# Cross-Validation

# What It Is

  • Technique to estimate model generalization by splitting data into train/validation folds
  • Reduces variance in performance estimates vs. single train/val split

# Common Strategies

Method Description Best For
K-Fold Split into K folds, rotate val set General use
Stratified K-Fold Preserves class distribution per fold Imbalanced classes
Leave-One-Out (LOO) K = N Small datasets
Time Series Split Forward-chaining folds, no future leakage Temporal data
Group K-Fold Keeps groups intact across folds Leakage-prone groups (e.g. patient IDs)

# Bias-Variance Tradeoff in CV

  • High K → low bias, high variance (expensive, folds very similar)
  • Low K → high bias, low variance
  • K=5 or 10 is the empirical sweet spot

# Nested CV

  • Outer loop: estimate generalization error
  • Inner loop: hyperparameter tuning
  • Prevents optimistic bias from tuning on the same split used for evaluation

# ⚠️ Tricky Interview Facts

  1. Data leakage in pipelines: preprocessing (e.g. scaling, SMOTE) must be fit inside each fold, not before CV — fitting on full data before CV leaks info into val folds
  2. LOO is not always best for small data: high variance between folds can make it less reliable than 5- or 10-fold despite using more training data
  3. Stratified split is needed for regression too (by binning target) when the target is skewed — plain K-Fold can create folds with very different distributions
