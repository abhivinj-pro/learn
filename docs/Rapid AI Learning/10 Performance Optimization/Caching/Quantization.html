Quantization

LLMs are trained in 32-bit or 16-bit floating point. Quantization compresses the model’s weights to lower bit-widths (8-bit, 4-bit, or even 2-bit), dramatically reducing memory footprint and inference latency. The core trade-off is precision vs. speed — lower bits means a smaller, faster model, but with potentially degraded output quality.

# Visual Overview — Bit-Width Comparison

FP32 32 ~280 GB (70B) Slow Training

FP16 / BF16 16 ~140 GB Moderate Fine-tuning

INT8 8 ~70 GB Fast Production

INT4 (GPTQ/AWQ) 4 ~35 GB Very fast Edge / serving

2-bit / 1-bit 1–2 ~9 GB Extremely fast Research only

Quality legend: = full = partial = degraded

Speculative decoding: a small draft model generates tokens; the large model verifies in parallel — combines both worlds

# Quantization Methods

# Post-Training Quantization (PTQ)

Applied after training with no retraining required. Fast to deploy, but can suffer quality loss on layers with outlier activations. Suitable when speed-to-deploy is a priority.

# Quantization-Aware Training (QAT)

The model is trained with simulated quantization noise injected during forward passes. Weights are “prepared” for lower precision from the start. Better output quality than PTQ, but requires a full training run.

# GPTQ

A calibration-based method that minimizes the reconstruction error of each layer’s output when quantizing to 4-bit. Operates layer-by-layer using a small calibration dataset. The most widely used 4-bit format for self-hosted LLMs.

# AWQ (Activation-aware Weight Quantization)

Identifies the most important weights by observing activation magnitudes, and protects them from aggressive quantization. Produces better quality than GPTQ at the same bit-width, especially on instruction-following tasks.

# GGUF (llama.cpp format)

A file format that supports mixed-precision quantization — different layers can use different bit-widths. Optimized for CPU inference and consumer hardware. Widely used for local deployment.

# Speculative Decoding

Speculative decoding combines a small draft model with a large target model:

  1. A small, quantized draft model rapidly generates a candidate sequence of tokens.
  2. The large target model verifies all candidate tokens in a single parallel forward pass.
  3. Correct tokens are accepted; the first incorrect token is rejected and regenerated.

This delivers 2–4× throughput improvement with zero quality degradation from the target model’s perspective, since the target model still controls every accepted token.

# Best Practices for Agentic Systems

  • Use INT8 for latency-sensitive production serving — the quality loss is negligible for most agentic tasks.
  • Use INT4 (AWQ) for memory-constrained deployments or when running multiple agent instances in parallel on the same GPU.
  • Reserve FP16 for fine-tuning or when output quality is critical and latency is not the bottleneck.
  • Pair quantization with speculative decoding for maximum throughput in high-concurrency agentic pipelines.

# Impact

Quantization to INT8 typically delivers 1.5–2× throughput improvement over FP16 at negligible quality cost. INT4 can fit a 70B model on a single 40GB GPU, enabling self-hosted deployments that would otherwise require multi-GPU clusters.
