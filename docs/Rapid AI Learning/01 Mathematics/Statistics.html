# 1. Normal Distribution (Gaussian Distribution)

PDF:

f(x)=σ2π​1​exp(−2σ2(x−μ)2​)
Symbol Meaning
μ Mean (center of the bell curve)
σ Standard deviation (spread/width)
σ2 Variance

What it means:

  • Symmetric, bell-shaped curve centered at μ.
  • ~68% of data falls within μ±σ, ~95% within μ±2σ, ~99.7% within μ±3σ (the 68-95-99.7 rule).
  • Fully described by just two parameters: μ and σ.

Notation: X∼N(μ,σ2)

Why it matters in ML:

  • Assumed in linear regression residuals, weight initialization (e.g. Xavier/He init), Gaussian Naive Bayes, and LDA.
  • Central Limit Theorem guarantees that sample means converge to a normal distribution — which justifies many statistical methods.

# 2. Mean, Median & Mode

# Mean (Arithmetic Average)

xˉ=n1​i=1∑n​xi​
  • The sum of all values divided by the count.
  • Sensitive to outliers. One extreme value can pull it far from the centre.
  • Used in virtually all ML algorithms (loss minimisation, gradient descent steps, etc.).

# Median

  • The middle value when data is sorted. For even n, it is the average of the two middle values.
  • Robust to outliers. Preferred over mean when data is skewed (e.g. income data).
  • In a normal distribution: mean=median=mode.

# Mode

  • The most frequently occurring value in a dataset.
  • Can have multiple modes (bimodal, multimodal) or none (uniform distribution).
  • Used in categorical data analysis and clustering to find cluster centres.

Quick comparison:

Sensitive to Outliers? Works on Categorical Data?
Mean ✅ Yes ❌ No
Median ❌ No ❌ No
Mode ❌ No ✅ Yes

# 3. Exponential Distribution

PDF:

f(x;λ)=λe−λx,x≥0
Symbol Meaning
λ Rate parameter (how fast events occur)
1/λ Mean (expected waiting time)

What it means:

  • Models the time between independent events that occur at a constant average rate λ.
  • Examples: time between server requests, time until a component fails, inter-arrival times.

Key properties:

Mean=λ1​,Variance=λ21​
  • Memoryless property: P(X>s+t∣X>s)=P(X>t). The past gives no information about the future — it has “no memory.”

Notation: X∼Exp(λ)

Why it matters in ML:

  • Used in survival analysis, Poisson processes, and queuing models.
  • Appears in regularisation: the Laplace prior (behind L1/Lasso regularisation) is a double exponential distribution.

# 4. Standard Deviation & Variance

# Variance

Population variance:

σ2=N1​i=1∑N​(xi​−μ)2

Sample variance (unbiased estimator):

s2=n−11​i=1∑n​(xi​−xˉ)2
  • Measures the average squared deviation from the mean.
  • Squaring makes all deviations positive and penalises large deviations more.
  • Units are squared (e.g. metres² for a dataset in metres) — less interpretable directly.

Why n−1? Dividing by n−1 (Bessel’s correction) corrects the bias that arises because the sample mean xˉ is used instead of the true population mean μ.

# Standard Deviation

σ=σ2​(population),s=s2​(sample)
  • Simply the square root of variance — restores the original units.
  • Directly interpretable: “on average, values are σ units away from the mean.”
  • A low σ → data is tightly clustered; a high σ → data is widely spread.

Why they matter in ML:

  • Feature scaling (standardisation): z=σx−μ​ transforms features to zero mean and unit variance.
  • Regularisation, Gaussian distributions, confidence intervals, and anomaly detection all depend on σ.

# 5. Population vs. Estimated (Sample) Parameters

In statistics and ML, you rarely have access to the entire population. You work with a sample and use it to estimate population parameters.

Concept Population Parameter Sample Estimator
Mean μ=N1​∑xi​ xˉ=n1​∑xi​
Variance σ2=N1​∑(xi​−μ)2 s2=n−11​∑(xi​−xˉ)2
Std. Dev. σ s
Size N (entire population) n (subset)

Key ideas:

  • A population parameter is a fixed, true (but usually unknown) quantity describing the whole group.
  • A sample statistic / estimator is computed from data you have and approximates the population parameter.
  • An estimator is unbiased if its expected value equals the true parameter: E[θ^]=θ.
  • As n→∞, sample statistics converge to population parameters — this is the Law of Large Numbers.

Maximum Likelihood Estimation (MLE):

θ^MLE​=argθmax​i=1∏n​P(xi​∣θ)

MLE finds the parameter θ^ that makes the observed data most probable. Note: MLE for variance uses n1​ (biased), which is why n−11​ is preferred in practice.

# 6. Hypothesis Testing

Hypothesis testing is a framework to decide whether observed data provides enough evidence to reject a default assumption. You never “prove” anything — you either find sufficient evidence against the null, or you don’t.

Think of it like a court trial: the defendant is innocent until proven guilty. The null hypothesis is innocent by default. Your data is the evidence.

# Null Hypothesis H0​

The default assumption — the boring, “nothing is happening” claim. It assumes no effect, no difference, no relationship.

H0​:μ=μ0​(or μ1​=μ2​, β=0, etc.)
  • You assume H0​ is true at the start.
  • You try to find evidence strong enough to reject it.
  • If evidence is weak, you fail to reject H0​ — you do NOT “accept” it. Absence of evidence is not evidence of absence.

Examples:

  • “The new model has the same accuracy as the baseline.” (μnew​=μold​)
  • “This feature has no effect on the target.” (β=0)
  • “The coin is fair.” (p=0.5)

# Alternative Hypothesis H1​ (or Ha​)

The challenger claim — what you’re trying to find evidence for. It represents a real effect, difference, or relationship.

H1​:μ=μ0​(two-tailed)
  • You never test H1​ directly — you only build evidence against H0​.
  • If H0​ is rejected, H1​ is supported (not proven).

Examples:

  • “The new model is more accurate than the baseline.” (μnew​>μold​)
  • “This feature does affect the target.” (β=0)
  • “The coin is biased.” (p=0.5)

# The Decision Framework

flowchart TD
  d["Collect data"] --> t["Compute test statistic<br/>(t, z, F, χ²)"]
  t --> p["Compute p-value"]
  p --> cmp{"Compare to significance level α"}
  cmp -->|"p ≤ α"| rej["Reject H₀<br/>evidence for H₁"]
  cmp -->|"p &gt; α"| fail["Fail to reject H₀<br/>not enough evidence"]

# p-value

p=P(observing this data, or more extreme∣H0​ is true)
  • A small p-value means: “if H0​ were true, this data would be very unlikely.” → Reject H0​.
  • A large p-value means: “this data is plausible under H0​.” → Don’t reject.
  • Common threshold: α=0.05 (5% significance level).

⚠️ The p-value is not the probability that H0​ is true. It’s the probability of your data given H0​ is true. A common interview trap!

# Significance Level α

  • Pre-chosen threshold for how much false-positive risk you’re willing to accept.
  • Typical values: 0.05 (standard), 0.01 (strict), 0.10 (lenient).

# Two Types of Errors

H0​ is actually True H0​ is actually False
Reject H0​ ❌ Type I Error (False Positive) ✅ Correct (True Positive)
Fail to Reject H0​ ✅ Correct (True Negative) ❌ Type II Error (False Negative)
  • Type I Error rate = α (significance level) — you control this directly.
  • Type II Error rate = β — harder to control; depends on sample size and effect size.
  • Power = 1−β = probability of correctly rejecting a false H0​. You want this high.

In ML terms: Type I = flagging a harmless input as anomalous. Type II = missing a real anomaly.

# One-Tailed vs. Two-Tailed Tests

Type H1​ Use when…
Two-tailed μ=μ0​ You care about differences in either direction
Right-tailed μ>μ0​ You only care if the new thing is better
Left-tailed μ<μ0​ You only care if the new thing is worse
  • Two-tailed tests split α across both tails (α/2 each side) — they are more conservative.
  • Always choose the tail before seeing data, or you risk p-hacking.

# Common Tests & When to Use Them

Test Use Case
z-test Known population σ, large n
t-test Unknown σ, small n; comparing means
Chi-squared χ2 Categorical data; testing independence or goodness of fit
F-test / ANOVA Comparing variances or means across 3+ groups
A/B test Comparing two variants (essentially a two-sample t-test)

# The One-Liner for Interviews

“H0​ is the default ‘no effect’ claim. We assume it true and collect evidence against it. If the p-value falls below α, the data is too unlikely under H0​ and we reject it in favour of H1​. We control Type I error with α, and aim for high power (1−β) to catch real effects.”

# 7. Covariance

# Definition

  • Measures linear relationship between two variables: how they vary together
  • Cov(X,Y)=E[(X−μX​)(Y−μY​)]
  • Sample: n−11​∑i=1n​(xi​−xˉ)(yi​−yˉ​)

# Sign & Magnitude

Value Meaning
> 0 X and Y increase together
< 0 X increases as Y decreases
= 0 No linear relationship
Large |val| Stronger linear co-movement

# Covariance vs Correlation

  • Correlation = normalized covariance: ρ=σX​σY​Cov(X,Y)​∈[−1,1]
  • Covariance is scale-dependent; correlation is not

# Covariance Matrix (Multivariate)

  • Σij​=Cov(Xi​,Xj​); diagonal = variances
  • Always symmetric and positive semi-definite
  • Used in PCA, Gaussian distributions, Kalman filters, Mahalanobis distance

# Key Properties

  • Cov(X,X)=Var(X)
  • Cov(aX,bY)=ab⋅Cov(X,Y)
  • Var(X+Y)=Var(X)+Var(Y)+2Cov(X,Y)
  • Independent → Cov = 0, but Cov = 0 ≠ independent

# ⚠️ Tricky Interview Facts

  1. Zero covariance ≠ independence. Y=X2 with X∼N(0,1): Cov(X,Y)=0 yet perfectly dependent.
  2. Covariance matrix of whitened data = Identity matrix — commonly assumed in naive Bayes and speeds up optimization.
  3. Sample covariance uses n−1 (Bessel’s correction) to give an unbiased estimator — using n underestimates population variance.

# 8. Correlation

# Definition

  • Measures linear relationship strength and direction, scale-invariant
  • Pearson: ρ=σX​σY​Cov(X,Y)​∈[−1,1]

# Types

Type Use Case Metric
Pearson Linear, continuous ρ
Spearman Monotonic, ordinal/non-normal Rank-based ρ
Kendall’s τ Small samples, ordinal Concordant/discordant pairs
Point-Biserial One binary, one continuous Special case of Pearson

# Key Properties

  • ρ=±1 → perfect linear relationship
  • ρ=0 → no linear relationship (non-linear may still exist)
  • Invariant to linear transformations: ρ(aX+b, cY+d)=ρ(X,Y) (if ac>0)
  • Symmetric: ρ(X,Y)=ρ(Y,X)

# Correlation Matrix

  • Diagonal always = 1
  • Symmetric, positive semi-definite
  • Used in feature selection, multicollinearity detection (VIF), portfolio theory

# Correlation vs Causation

  • Correlation never implies causation — confounders, reverse causality, spurious correlation

# ⚠️ Tricky Interview Facts

  1. Pearson is sensitive to outliers — a single outlier can flip sign; Spearman is robust because it uses ranks.
  2. Correlation = 0 doesn’t mean independence — same as covariance; Y=X2, X∼N(0,1) has ρ=0 but strong dependence.
  3. Pearson correlation = cosine similarity on mean-centered vectors — directly links linear algebra to statistics.
  4. A small p-value means: your observed correlation would be rare if there were truly no relationship.
  5. If there were really no relationship at all in the population, what is the probability of getting a sample correlation this strong or stronger just by random chance?

# 9. Standard Error (SE)

# Definition

  • SE = standard deviation of a sampling distribution of a statistic (usually mean)

# Formula

Population SE:

SE=n​σ​

Sample estimate:

SE=n​s​

# Key Intuition

  • Measures how precisely a sample statistic estimates the population parameter
  • Smaller SE ⇒ more reliable estimate
  • SE decreases as sample size increases

# Common Use Cases

  • Confidence intervals:
xˉ±z⋅SE
  • Hypothesis testing (z-test, t-test)
  • Comparing model performance estimates

# SE vs Standard Deviation

Concept Standard Deviation (SD) Standard Error (SE)
Measures Data spread Estimate uncertainty
Depends on Data variance Variance + sample size
Formula σ or s σ/√n or s/√n

# In ML Context

  • Used in cross-validation to report variability of metrics
  • SE of mean accuracy across k folds:
SE=k​s​(k=folds)

# Tricky Interview Facts

  • SE ≠ SD; confusing them is a common mistake
  • Increasing data size reduces SE but not SD
  • For small samples, use the t-distribution instead of normal for confidence intervals
